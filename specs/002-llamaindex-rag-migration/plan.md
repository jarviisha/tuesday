# Implementation Plan: LlamaIndex End-to-End RAG Migration

**Branch**: `[002-llamaindex-rag-migration]` | **Date**: 2026-04-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-llamaindex-rag-migration/spec.md`

## Summary

Migrate the `src/tuesday/rag/` capability layer từ custom HTTP-based providers
và shim-only LlamaIndex bridge sang LlamaIndex's full ecosystem end-to-end:
node parsing, embedding, vector store (Qdrant), retrieval, và LLM generation.
FastAPI shell, public HTTP contract, domain Protocols, và use-case orchestration
được giữ nguyên. Toàn bộ locked semantics (DL-001 đến DL-009) được re-verify
hoặc re-implement tại infrastructure adapter boundary.

**Hard prerequisite**: Python 3.12 phải được cài trước khi bắt đầu bất kỳ
implementation nào. LlamaIndex 0.14.x không import được trên Python 3.14.

## Technical Context

**Language/Version**: Python 3.12 (downgrade từ Python 3.14 hiện tại — R-001)
**Primary Dependencies**: FastAPI, Pydantic, llama-index-core 0.14.x,
  llama-index-vector-stores-qdrant, llama-index-embeddings-{openai,gemini,azure-openai},
  llama-index-llms-{openai,gemini,azure-openai}, qdrant-client, pytest, ruff
**Storage**: Qdrant — collection prefix mới `tuesday_v2__`; collections cũ
  (`tuesday__`) giữ nguyên cho đến khi xóa thủ công
**Testing**: pytest — unit, api, integration, smoke, regression
**Target Platform**: Linux server, Python 3.12+
**Project Type**: Capability-oriented web service
**Performance Goals**: Retrieve + generate latency trong vòng 20% so với
  baseline trước migration (SC-003)
**Constraints**: Không thay đổi public API contract; domain Protocols giữ
  nguyên; `TUESDAY_*` env vars được preserve; demo mode không cần API keys
**Scale/Scope**: Thay toàn bộ `rag/infrastructure/` adapters và
  `runtime/container.py` wiring; use cases và API layer không đụng vào

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

### Principle I — Preserve Existing Architecture
**Status**: PASS với justified exception.

Boundary split (`api/`, `domain/`, `ingestion/`, `retrieval/`,
`generation/`, `infrastructure/`) được giữ nguyên. LlamaIndex types chỉ
sống trong `infrastructure/`. Domain Protocols (`VectorStore`,
`EmbeddingProvider`, `LLMProvider`, `Chunker`) được giữ làm contracts
giữa các layer.

**Justified exception**: Việc thay toàn bộ `infrastructure/` adapters là
yêu cầu của feature spec — không phải speculative. Mỗi custom adapter
đang bị thay đều có LlamaIndex equivalent trực tiếp.

### Principle II — Minimal Diff Delivery
**Status**: JUSTIFIED EXCEPTION.

Đây là capability-layer rewrite, không phải surgical fix. Piecemeal
LlamaIndex integration đã được evaluate và reject vì nó để lại project
trong trạng thái "partially used" — đúng vấn đề đang motivation migration.
Spec explicitly authorizes phạm vi này.

### Principle III — Tests for Changed Behavior
**Status**: PASS.

Mọi locked semantic đều có test. Thay đổi chunking (DL-006 superseded)
yêu cầu regenerate golden cases. Toàn bộ locked semantics khác giữ
existing tests với adapter-boundary adjustments.

### Principle IV — Public Contract and Schema Stability
**Status**: PASS với documented exception.

HTTP contract: **không đổi**.
Qdrant storage schema: **thay đổi**. LlamaIndex's `QdrantVectorStore` dùng
payload format khác với bridge hiện tại. Documented trong `data-model.md`.
New collection prefix (`tuesday_v2__`) tránh silent data corruption.
Collections cũ cần re-index.

### Principle V — Follow Current Conventions
**Status**: PASS.

Python 3.12, type hints, `snake_case`/`PascalCase`, English, ruff config
không đổi.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|---|---|---|
| Broad `infrastructure/` replacement | LlamaIndex E2E yêu cầu thay toàn bộ custom HTTP adapters | Piecemeal integration giữ nguyên vấn đề "partially used" dependency |
| Qdrant storage schema thay đổi | LlamaIndex QdrantVectorStore dùng payload format riêng | Fork llama-index-vector-stores-qdrant để match schema cũ = long-term maintenance burden |
| DL-006 (character chunking) superseded | Token-based chunking là LlamaIndex default, mang lại semantic splitting value | Character-mode trong SentenceSplitter tồn tại nhưng mất đi giá trị của migration |
